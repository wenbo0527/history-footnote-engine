## 📊 详细审计

### 1. 类型旧字段(5 个)

**Archive type (types.ts:322-323)**:
```typescript
// ❌ 旧字段,后端不返回

```

**仍被引用**:
- `ArchiveCard.svelte:7-41` - 派生函数优先用旧字段(向后兼容)
- 5 处派生逻辑都加了兜底

**修法建议**:
- 后端 v2.11 移除返回
- 前端 v2.11 移除 `character_name?`/`character_occupation?` 类型
- 当前向后兼容是合理的

### 2. as any 类型绕过(46 处产线 / 13 处测试)

**典型案例**:
```typescript
// FateCardDetailModal.svelte:47-48
if ((res as any).updated_state) {
  gameActions.set((res as any).updated_state);
}
// 修法：在 fateUse() 返回类型加 updated_state?: GameState
```

**分布**:
- API 响应处理: 23 处
- 组件跨类型: 12 处
- 渲染兜底: 11 处

**根因**:
- 后端响应类型未严格定义
- 前端组件 props 类型不完整
- 测试中需要绕过类型

**修法**:
- 用 `unknown` 代替 `any` + 显式 narrowing
- 加 `as Type` 精确断言而非 `as any`
- 写 mapper 函数统一转换
