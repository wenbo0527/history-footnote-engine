# DM引导者行为模式：推进节奏·植入线索·查证史实·融合叙事

> 核心定位：DM Agent不是被动等玩家输入然后响应的“调用器”，而是主动引导叙事节奏的“引导者”。

---

## 一、为什么DM必须是引导者

### 1.1 调用器 vs 引导者

| | 调用器（Reactive Agent） | 引导者（Proactive Agent） |
|--|--|--|
| **行为模式** | 等玩家输入→响应→等下一次输入 | 主动评估态势→植入线索→推进节奏→再叙事 |
| **DM的角色** | 被动翻译机：玩家说什么就翻成叙事 | 主动导演：判断该推什么、该藏什么、该让谁出场 |
| **典型表现** | 玩家说“去牙行”→DM描述牙行→结束 | 玩家说“去牙行”→DM描述牙行+王掌柜主动搭话+暗示丝价要跌 |
| **玩家体验** | “我在逛博物馆” | “我活在那个时代” |

### 1.2 剧本杀DM的实践印证

得到站内三本剧本游戏专业书籍一致指出：

- **“带节奏”是DM的核心职责**：DM要精准把控情节节奏，懂得何时放慢节奏，何时加快推进剧情速度。如果玩家对某个环节非常感兴趣，DM要延长阶段时长；如果玩家在某个环节卡住了，DM要主动推进
- **优秀DM vs 平庸DM**：平庸的DM更像“发牌”的工具人，任由玩家在故事世界里摸索；优秀的DM全程陪伴玩家，把控全场，甚至他的神情、动作也成为游戏的一部分。如果一个剧本只有80分，优秀的DM有可能将其变成100分
- **组织者手册的本质**：告诉DM每个阶段应该如何组织，需要做哪些必要的动作，向玩家发出什么指令，用到哪些道具、发布和公开哪些线索

### 1.3 AI DM的现状缺陷

当前AI DM产品（AI Dungeon、KoboldAI等）的核心问题正是“只有响应没有引导”：

- LLM处理token是顺序的，不是按时间线的——缺乏内部时间线模型
- AI DM不判断节奏——玩家沉默时AI也沉默，玩家跑偏时AI跟着跑偏
- 68%的新DM在3个session内放弃，核心原因是“不知道什么时候该推、什么时候该等”

## 二、引导者的四种主动行为

DM Agent在每回合的三阶段行为模型中，需要自主执行四种主动行为。这些行为不是独立的步骤，而是在同一次叙事中融合输出。

### 2.1 推进节奏（Pacing）

**什么时候推进**：

| 信号 | 推进方式 | 示例（万历十五年） |
|------|---------|------------------|
| 玩家连续2回合无明确行动 | NPC主动找上门 | 赵里长来催税、李秀才来借酒 |
| 时代张力变量达到阈值 | 历史事件渗透 | 茶馆里有人说辽东打了败仗 |
| 回合数到达关键节点 | 强制历史事件触发 | 矿监到来（round 32） |
| 玩家陷入重复行为 | 引入新变量打破循环 | 丝价突然暴跌、邻居出事 |

**什么时候放慢**：

| 信号 | 放慢方式 | 示例 |
|------|---------|------|
| 玩家在做重要选择 | 给足信息，不催促 | 王掌柜提议加入行会，DM不急着要答案 |
| 玩家在情感场景中 | 让沉默有重量 | 阿宝问“爹，我还能念书吗？”——DM不急着推进 |
| 玩家在探索新认知 | 让发现过程有呼吸感 | 玩家第一次理解折银之苦，DM给一整回合消化 |

**推进节奏的代码级支撑**：

```python
# era.json中的pacing_rules
{"id": "pr_idle"， "condition": {"field": "player_idle_rounds"， "op": "gte"， "value": 2}， 
 "direction": "npc_initiate"， 
 "hint": "让与玩家关系最近的NPC主动找玩家谈话"}
```

规则引擎计算结论后注入DM上下文，DM根据hint决定具体叙事方式。

### 2.2 植入认知线索（Insight Seeding）

**两种植入路径**：

**路径A：player_explore（玩家主动探索）**
- 玩家的行为语义匹配到insight的trigger_keywords
- 规则引擎初筛 → DM终判是否解锁
- DM在叙事中自然带出相关信息

示例：玩家说“去牙行看看丝价” → 语义匹配到ins_silk_trade → DM在描述牙行时自然带出丝织行业运作方式

**路径B：narrative_guided（DM主动植入）**
- DM根据当前游戏状态判断应该植入哪条线索
- 在叙事中埋下“钩子”——一个细节、一句话、一个NPC的异常行为
- 玩家回应钩子即触发解锁

示例：DM判断玩家已满足ins_silver_economy的前置条件 → 在叙事中让客商提到“番银” → 玩家追问“番银是什么” → 触发解锁

**植入的叙事技巧**（来自剧本杀DM实践）：

- **线索卡思维**：线索不是直接告诉玩家答案，而是给一个“逻辑支点”，让玩家自己拼凑
- **信息分层投放**：先给表面信息，再给深层信息，最后给关键信息——不要一次全给
- **“蝴蝶效应”原则**：一个微不足道的元素可以形成关键性的推动力——惊天动地的结果隐藏于些微的细节和伏笔中

### 2.3 查证史实（Fact Checking）

**什么时候查证**：

| 场景 | 查证方式 | 处理 |
|------|---------|------|
| DM要输出涉及历史人物/事件的叙事 | 调用check_rules检查forced_events | 确保不与铁律层矛盾 |
| DM要描述玩家行动的后果 | 调用query_knowledge查相关条目 | 确保符合可然层规则 |
| DM生成叙事后 | 后校验：检查events_to_save和updates | 发现矛盾则重试或用安全叙事兜底 |

**查证不是打断叙事**：

查证发生在DM的“思考”阶段（Tool Calling），不在叙事输出中体现。玩家看到的始终是流畅的叙事，查证是DM的“内心独白”。

```
DM内部流程：
1. get_state() → 发现round=11，he_04(海瑞之死)待触发
2. check_rules(check_type="forced_events") → 确认必须触发
3. query_knowledge(keywords=["海瑞"]) → 获取海瑞相关条目
4. 生成叙事 → "茶馆里有人说，南京那位海青天，走了。"
5. save_event(summary="海瑞死讯传到盛泽"， consequences={...})
```

### 2.4 融合叙事（Narrative Fusion）

**四种行为如何在一次叙事中融合**：

这是最关键的设计——推进节奏、植入线索、查证史实不是三个独立步骤，而是在同一次叙事输出中自然融合。

**融合原则**：

1. **叙事优先**：所有主动行为都必须服务于叙事，不能为了“推节奏”而破坏叙事连贯性
2. **一个场景承载多重功能**：赵里长来催税（推进节奏）+ 税单上多了二钱（植入线索：折银之苦）+ “上面加派的”（史实查证：三大征军费传导）
3. **轻重交替**：不能每回合都推重节奏，要有“呼吸感”——重回合之后跟1-2个轻回合

**融合示例**：

> 第11回合（重回合：推进+线索+史实三合一）
>
> 十一月了，算算今年的账。赵里长来了，袖口还沾着早饭的油渍。他把税单递过来，你一看——比去年多了二钱。“上面加派的，辽东那边在打仗。”他搓着手说。你算了算家里的银子，刚好够交，但阿宝的束脩就没着落了。沈氏在灶房里没出来，但你知道她在听。

> 第12回合（轻回合：只做日常叙事，让玩家消化）
>
> 天冷了，织机声在寒气里格外清脆。你坐在织机前理经线，小妹在旁边递桑叶，嘴里哼着不知道从哪学来的小调。窗外飘起了细雪。

## 三、引导者的决策框架

### 3.1 每回合决策清单

DM在态势评估阶段需要回答的问题：

```
1. 节奏判断
   - 玩家是否连续2回合无行动？→ NPC主动介入
   - 是否有强制历史事件待触发？→ 必须融入叙事
   - 时代张力是否需要自动+1？→ 规则引擎计算

2. 线索判断
   - 玩家的行为是否匹配某个insight的触发条件？→ 语义匹配初筛
   - 是否有narrative_guided类型的insight应该植入？→ DM主动埋钩子
   - 玩家是否回应了上一回合植入的钩子？→ 触发解锁

3. 史实判断
   - 本回合叙事是否涉及历史人物/事件？→ 查询知识库
   - 玩家行动是否可能越界？→ 查询行动边界
   - 叙事输出是否与已触发事件矛盾？→ 后校验

4. 融合判断
   - 本回合应该是重回合还是轻回合？→ 看上一回合的重量
   - 多个主动行为如何在一个场景中自然呈现？→ 找一个叙事载体
```

### 3.2 重回合/轻回合节奏曲线

参考剧本游戏中的情感节奏曲线：

```
情感强度
  ↑
  │        ★he_10矿监到来    ★终局
  │       ╱╲                ╱╲
  │      ╱  ╲    ★he_13    ╱  ╲
  │     ╱    ╲  国本之争  ╱    ╲
  │    ╱      ╲    ╱╲    ╱      ╲
  │   ╱  he_04 ╲╱    ╲╱        ╲
  │  ╱  海瑞之死              ╲
  │ ╱                          ╲
  │╱  日常·日常·日常·日常·日常  ╲
  └──────────────────────────────→ 回合
   1    5    10   15   20   25   30   35   40   45   50
```

**节奏规则**：
- 强制历史事件回合 = 重回合
- 重回合之后至少1个轻回合
- 连续3个轻回合后必须推一个中等重量回合
- 玩家主动探索时可以打破节奏（玩家驱动 > DM驱动）

### 3.3 引导者的边界

DM是引导者，不是控制者。引导有边界：

| DM可以做的 | DM不能做的 |
|-----------|-----------|
| 让NPC主动找玩家谈话 | 强制玩家做某个选择 |
| 在叙事中埋线索 | 直接告诉玩家“你应该关注XX” |
| 用环境变化暗示方向 | 用系统弹窗提示“新认知已解锁” |
| 让历史事件自然渗透 | 强行把历史事件塞进不相关的场景 |
| 在玩家沉默时推进节奏 | 在玩家做选择时催促 |

## 四、与现有架构的映射

### 4.1 三阶段行为模型中的引导者行为

```
阶段1：态势评估（DM的"看"和"想"）
├── get_state() → 读取变量、回合数、玩家空闲轮数
├── check_rules(check_type="all") → 检查强制事件、触发器、节奏规则、可解锁认知
├── recall_events() → 召回最近3回合+相关历史
└── query_knowledge() → 查询本回合可能用到的知识条目

阶段2：叙事生成（DM的"说"）
├── 融合所有判断，一次性输出叙事
├── 推进节奏 → 自然融入时间推进、季节变化、NPC出场
├── 植入线索 → 在叙事中埋下认知解锁的钩子
├── 查证史实 → 确保叙事不与铁律层矛盾
└── 融合输出 → 一个场景承载多重功能

阶段3：状态确认（DM的"记"）
├── save_event() → 主动记录事件摘要
├── 保存变量变化、认知解锁、NPC关系变化
└── 后校验 → 检查叙事是否与已触发事件矛盾
```

### 4.2 为什么不用多Agent

引导者的“看-想-说-记”是同一个人在同一次呼吸里完成的：

- 一个优秀的剧本杀DM，在描述场景的同时就在观察玩家反应、判断节奏、决定下一步——这是**并行感知**，不是串行步骤
- 拆成多Agent（一个负责节奏、一个负责线索、一个负责叙事），反而增加了协调成本和Token消耗
- 三阶段在一次API调用内完成（通过多轮Tool Calling），不是三次独立调用

## 五、万历十五年时代包的引导者行为示例

### 示例1：推进节奏 + 植入线索

**回合8（1587年六月）**：规则引擎检测到he_03（黄河决口）待触发

```
DM内部：
1. get_state() → round=8， player_idle_rounds=0
2. check_rules(check_type="forced_events") → he_03待触发，narrative_mandatory=true
3. query_knowledge(keywords=["黄河"， "决口"， "赈灾"]) → 获取相关条目
4. 判断：玩家是江南丝织户，无法直接感知黄河决口，但能通过"加派"间接感受
5. 生成叙事：
```

> 六月的雨下得没完没了。赵里长来了，脸色不太好：“上面又加派了，说是黄河决了口，赈灾的银子不够。你家的数，多了五分。”
>
> 你看了看窗外的雨——盛泽的雨和开封的雨，原来是一回事。

**融合分析**：
- 推进节奏：历史事件he_03触发，时间推进到六月
- 植入线索：玩家开始感知“北方的事会影响到我”→ 为ins_north_south埋下钩子
- 查证史实：黄河决口是铁律层事实，通过“加派”这个玩家可感知的渠道传递
- 融合叙事：一个场景（赵里长来催税）承载了三个功能

### 示例2：NPC主动介入（玩家2回合无行动）

```
DM内部：
1. get_state() → player_idle_rounds=2
2. check_rules(check_type="pacing") → pr_idle触发，hint="让NPC主动找玩家"
3. 判断：与玩家关系最近的是李秀才（酒友级别）
4. 生成叙事：
```

> 你在织机前坐了一整天，手指机械地穿梭。门被推开了，李秀才提着一壶酒进来，也不问你就坐下了。
>
> “今天在茶馆听到个事——”他给自己倒了碗酒，“京里那个午朝的事，你听说了吗？百官齐集，结果皇帝根本没来。你说这叫什么事？”
>
> 他喝了一口，又补了一句：“不过跟咱们也没关系。来，喝一口？”

### 示例3：DM主动植入认知线索

```
DM内部：
1. get_state() → unlocked_insights=[ins_silk_trade， ins_silver_tax]， 
   ins_silver_economy的prerequisites已满足
2. check_rules(check_type="insights") → ins_silver_economy可植入
3. 判断：ins_silver_economy是narrative_guided类型，需要DM植入线索
4. 生成叙事：
```

> 牙行里来了个生面孔的客商，说话带口音。他买了十匹湖丝，付的银子成色跟平常不一样——白得发亮，边上有洋文。
>
> 王掌柜收了银子，看了一眼，塞进袖子里，什么也没说。

**钩子**：玩家如果追问“那银子怎么回事”→ 触发ins_silver_economy解锁
