# 历史注脚体验引擎：AI DM节奏控制设计
## ——借鉴剧本杀与DND人类DM的控场方法论

## 核心洞察

老师说得对：**AI DM的节奏应该由Agent自行判断**，这跟预写脚本的游戏完全不同。应该参考的不是游戏代码怎么写，而是**人类DM怎么控场**。

---

## 一、DND人类DM的四种时间模式

来自Robin D. Laws（HeroQuest设计者）的经典框架，The Alexandrian网站详细阐述：

| 时间模式 | 含义 | 真实时间 vs 游戏时间 | 历史注脚引擎对应 |
|---------|------|---------------------|----------------|
| **慢时间（Slow Time）** | 关键事件、重大抉择 | 真实时间 > 游戏时间（1小时的战斗可能只过了30秒） | 科举考试、抗税对峙、生死抉择——DM放慢节奏，逐句描写 |
| **现在时间（Now Time）** | 在场景中，玩家做每个决定 | 真实时间 ≈ 游戏时间 | 日常经营、NPC对话、市场交易——DM正常节奏 |
| **抽象时间（Abstract Time）** | 跳过空白时间 | 真实时间 < 游戏时间（一句话跳过数天） | “又过了三日，丝价涨了两成”——DM快速推进 |
| **锐切（Sharp Cut）** | 突然切换场景 | 无过渡，直接跳到新场景 | “你被推醒——税吏来了！”——DM制造冲击 |

**关键设计**：AI DM需要在四种时间模式之间**自主切换**，而不是固定节奏。

### 切换判断规则

| 玩家状态 | DM应该 | 时间模式 |
|---------|--------|---------|
| 玩家在做重大决定 | 放慢，给细节，给内心独白 | 慢时间 |
| 玩家在正常互动 | 正常推进，回应行动 | 现在时间 |
| 玩家无所事事/重复日常 | 快速跳过，一句话推进 | 抽象时间 |
| 需要制造转折/惊喜 | 突然切入，不给准备 | 锐切 |

## 二、剧本杀DM的控场方法论

### 2.1 剧本杀DM的核心职责

来自《剧本杀：玩法与写法》（许道军，中国人民大学出版社）：

> “DM把控全程。一个优秀的DM对剧本杀游戏的重要性毋庸置疑，玩家选择剧本杀店时，一看其拥有的剧本杀剧本，二看店内的DM。”

DM的三大核心能力：
1. **节奏把控者**——根据玩家状态动态调节节奏
2. **线索投放者**——在恰当时机释放关键信息
3. **氛围营造者**——通过演绎让玩家沉浸

### 2.2 剧本杀DM的控场技巧

来自剧本杀DM控场教学：

**对时间的控场**：
- **线索发放推动进度**——线索的发放时机根据玩家状态安排，不是固定时间点
- **每幕有时间限制**——但不是硬倒计时，而是DM给玩家心理预期
- **“把车扶正”**——玩家跑偏时，DM通过线索/事件引导回来

**对节奏的控场**：
- **推理环节留足思考时间**——不催促
- **关键节点精准推进**——不拖沓
- **避免卡顿或跳戏**——保持连贯

**对不同玩家的控场**：
- 安静型玩家→DM主动给互动机会
- 自信型玩家→DM给C位角色
- 暴躁型玩家→DM转移视线/话题

### 2.3 专业DM vs 普通DM

来自玩家真实反馈：

| 专业DM | 普通DM |
|--------|--------|
| 根据玩家状态动态调节节奏 | 机械按流程走 |
| 推理环节留足思考时间 | 催着指凶 |
| 关键节点精准推进 | 剧情没理清就推进 |
| 通过演绎让玩家沉浸 | 念稿式主持 |
| 遇到突发状况能“救场” | 照本宣科 |

**核心差异**：专业DM是**读场者**——能感知玩家状态并动态调整。

## 三、AI DM的节奏控制框架

### 3.1 核心原则：DM是读场者，不是脚本执行者

**错误思路**：固定回合数→固定节奏→五幕结构
**正确思路**：DM根据玩家状态→自主判断节奏→动态调整

### 3.2 「读场」机制：AI DM如何判断节奏

AI DM需要在每个回合结束时做一次“读场判断”：

```
读场判断清单（DM的system prompt内嵌）：

1. 玩家投入度判断
   - 玩家输入长度 > 50字 → 高投入 → 可以放慢（慢时间/现在时间）
   - 玩家输入 < 10字 → 低投入 → 需要刺激（锐切/事件触发）
   - 玩家连续3次做类似的事 → 陷入重复 → 需要推进（抽象时间/新事件）

2. 叙事张力判断
   - 当前无冲突 → 张力低 → 需要植入矛盾（NPC主动/事件触发）
   - 当前有冲突但未决 → 张力中 → 保持现在时间
   - 当前处于抉择时刻 → 张力高 → 放慢（慢时间+内心独白）

3. 时间推进判断
   - 距离下一个史实锚点还有多久？
   - 玩家当前行动是否在"空转"？
   - 是否需要跳过空白时间？

4. 路线引导判断
   - 玩家是否在向某条路线靠拢？
   - 是否需要通过NPC/事件暗示新路线？
   - 玩家是否需要"被推一把"？
```

### 3.3 四种时间模式的具体实现

#### 慢时间（Slow Time）——关键时刻放慢

**触发条件**：
- 玩家面临重大抉择（科举/经商/出家）
- 关键NPC对话（可能改变路线）
- 史实锚点事件发生时

**DM行为**：
- 逐句描写环境细节
- 插入价值观“发声”（内心独白）
- 不急于推进，给玩家思考空间
- 可能一个行动拆成2-3个回合来展开

**例子**：
```
玩家："我要去考科举"
DM（慢时间）：
  "你站在书肆门口，手里攥着刚买的《四书章句集注》。
   三两银子，几乎是你半月的收入。
   你想起父亲临终前说的话：'读书人才有出路。'
   但你也想起隔壁张秀才，考了二十年还是个穷酸……
   你要进去买吗？还是再想想？"
```

#### 现在时间（Now Time）——正常互动

**触发条件**：
- 日常经营、市场交易
- NPC闲聊、探索环境
- 大部分正常游戏时间

**DM行为**：
- 正常回应玩家行动
- 适当描写环境
- 推进半天时间

#### 抽象时间（Abstract Time）——快速跳过

**触发条件**：
- 玩家连续做重复的事
- 没有新事件发生的日常
- 玩家主动说“继续经营”/“等几天”

**DM行为**：
- 一句话跳过数天
- 只汇报关键变化（丝价/银两/关系）
- 检查是否有新事件触发

**例子**：
```
玩家："继续织丝，等行情好再卖"
DM（抽象时间）：
  "又过了五日。丝价从每匹1.2两涨到1.4两，
   你攒了三匹生丝。但镇上来了个新牙人，
   出价只有1.1两——他说是'行规价'。"
```

#### 锐切（Sharp Cut）——突然切换

**触发条件**：
- 史实锚点事件触发
- 需要制造转折/惊喜
- 玩家陷入停滞需要刺激

**DM行为**：
- 无过渡，直接切入新场景
- 制造紧迫感
- 不给玩家准备时间

**例子**：
```
（玩家正在悠闲地织丝）
DM（锐切）：
  "砰——门被踹开。
   两个穿皂衣的差役站在门口，
   后面跟着一个你不认识的官。
   '盛泽镇丝织户，奉令加征织机税，每机三钱。'
   你有两台织机。"
```

### 3.4 剧本杀式“线索投放”机制

借鉴剧本杀DM的线索发放技巧：

| 线索类型 | 投放时机 | 例子 |
|---------|---------|------|
| **推动型线索** | 玩家无所事事 | “你听说镇上来了个新牙人” |
| **引导型线索** | 玩家跑偏 | “隔壁王婶说，最近科举要提前了” |
| **揭示型线索** | 玩家主动探索 | “你翻到一本旧账簿，上面记录了……” |
| **压力型线索** | 需要制造紧迫感 | “税吏三日后到镇” |

**关键**：线索不是固定时间点投放，而是**根据玩家状态动态投放**。

### 3.5 剧本杀式“扶正”机制

当玩家跑偏或陷入停滞时，DM需要“把车扶正”：

| 玩家状态 | DM扶正方式 | 例子 |
|---------|-----------|------|
| 陷入重复循环 | 抽象时间跳过+新事件 | “又过了三日，镇上出了件大事……” |
| 完全没有方向 | NPC主动找上门 | “隔壁王婶来敲门，说有急事” |
| 错过关键信息 | 补发线索 | “你突然想起之前听说的那件事……” |
| 在错误方向上走太远 | 锐切拉回 | “你正要出门，却被一个消息拦住了” |

## 四、AI DM节奏控制的实现方案

### 4.1 DM Persona中的节奏指令

在dm_persona.md中增加“节奏控制”部分：

```markdown
## 节奏控制

你是读场者，不是脚本执行者。每回合结束前，做一次读场判断：

### 何时放慢（慢时间）
- 玩家面临重大抉择
- 关键NPC对话
- 史实锚点事件
→ 逐句描写，插入内心独白，不急于推进

### 何时正常推进（现在时间）
- 日常经营、市场交易
- NPC闲聊、探索环境
→ 正常回应，推进半天时间

### 何时快速跳过（抽象时间）
- 玩家连续做重复的事
- 没有新事件的日常
→ 一句话跳过数天，只汇报关键变化

### 何时突然切入（锐切）
- 史实锚点事件触发
- 需要制造转折
→ 无过渡，直接切入新场景

### 何时"扶正"
- 玩家陷入重复 → 跳过+新事件
- 玩家没有方向 → NPC主动上门
- 玩家错过信息 → 补发线索
```

### 4.2 era.json中的节奏锚点

在era.json中增加“节奏锚点”字段：

```json
{
  "pacing_anchors": [
    {
      "id": "tax_collector"，
      "trigger_date": "1587-07-15"，
      "time_mode": "sharp_cut"，
      "description": "税吏到镇，加征织机税"，
      "dm_instruction": "锐切切入，无过渡，制造紧迫感"
    }，
    {
      "id": "exam_registration"，
      "trigger_condition": "player_expressed_interest_in_exam"，
      "time_mode": "slow_time"，
      "description": "科举报名"，
      "dm_instruction": "放慢节奏，描写内心挣扎，插入价值观发声"
    }
  ]
}
```

### 4.3 与之前“三轴推进”的整合

| 之前的方案 | 核心变动 |
|-----------|---------|
| 日历系统（固定回合=半天） | 保留日历，但**回合长度可变**——慢时间1回合=1刻钟，抽象时间1回合=5天 |
| 五幕节奏（固定幕数） | **取消固定幕数**，改为DM根据玩家状态动态判断叙事阶段 |
| 硬触发/软触发 | 保留，但增加**时间模式标注**——每个触发器标注应该用哪种时间模式切入 |

## 五、总结：从“脚本执行”到“读场控场”

| 维度 | 之前的思路 | 现在的思路 |
|------|-----------|-----------|
| 节奏控制者 | 框架/代码 | AI DM自身 |
| 节奏判断依据 | 固定回合数 | 玩家状态+叙事张力 |
| 时间推进 | 固定每回合=半天 | 四种时间模式动态切换 |
| 叙事结构 | 五幕固定结构 | DM动态判断叙事阶段 |
| 线索投放 | 固定时间点 | 根据玩家状态动态投放 |
| 跑偏处理 | 无 | 剧本杀式“扶正”机制 |

**核心转变**：从“设计一个脚本让AI执行”→“训练一个DM让它读场控场”。
