## 一、产品定义

### 1.1 一句话定义

AI当DM，你当历史里的小人物。通过对话体验一个历史时期，历史走势不可改，但你的选择路径完全开放。

### 1.2 不是什么

- 不是"历史课"——没有知识点考核，只有沉浸体验
- 不是"架空冒险"——史实是铁律，不能改写
- 不是"经营模拟"——核心不是数值优化，是人在时代中的处境
- 不是"多Agent协作系统"——单Agent+Tool，最小架构
- 不是"游戏关卡"——**没有Game Over**，只有"故事转向"（v1.4+）
- 不是"单机游戏"——**支持多人并发**（v1.6+）

### 1.3 是什么

**配置驱动的AI叙事系统 + Disco Elysium风格认知驱动系统 + KV 缓存优化**（v1.6+）。

核心循环：
```
玩家输入 → 8 SKILL编排 → DM生成叙事 → 后校验+重试 → 规则引擎计算状态 → DM反馈结果
         ↑                       ↓
   （含 voice_options）  （KV 缓存节省 70-98%）
```

### 1.4 产品定位

**"受约束的自由"**——历史事实是铁律，但玩家的选择路径完全开放。

约束不是限制，是体验的来源：正因为你知道历史不可改，小人物的挣扎才有了重量。

市面上无同类产品。AI Dungeon 类无史实约束，DE 类无历史约束，Jomini Engine 是唯一学术先例（宏观战略视角），但无 AI DM 层。

## 三、模块设计

### 3.1 DM Agent（v1.6+ 完整版）

#### 3.1.1 5 阶段行为模型（v1.6+）

DM 每回合**5 阶段串行推理**（v1.4 是 4 阶段，v1.6 加入后校验阶段）：

```
阶段 0: 8 SKILL 编排（确定性逻辑归代码）
  → 输出 DMContext + skill_directive 文本

阶段 1: 态势评估（Reason about Situation）
  → DM 自主决定调用哪些 Tool

阶段 2: 叙事生成（Generate Narrative）
  → 必须输出 voice_options（2-4 个 DE 风格内在声音选项）
  → 输出 intent_type（action/inquire/describe/voice）

阶段 3: 🆕 后校验 + 重试（v1.6+）
  → post_validate 检查 5 层（格式/铁律/边界/时间/锚点）
  → error 级问题 → dm_agent.regenerate() 重新生成
  → 2 次重试仍失败 → generate_safe_narrative() 兜底

阶段 4: 状态确认（Confirm & Save）
  → 保存事件到记忆 + voice_options 持久化
```

#### 3.1.2 KV 缓存集成（v1.6+ 新增）

`dm_agent.py` 在 system prompt 上加 `cache_control={"type": "ephemeral"}`：

```python
system_message = SystemMessage(
    content=[{
        "type": "text",
        "text": system_prompt,
        "cache_control": {"type": "ephemeral"},  # ← 5 分钟 TTL
    }]
)
```

**节省估算**（50 回合实测）：
- 单回合 input tokens：~5500 → ~1200（**78% ↓**）
- Cache 命中率：50 回合下 **98%**（首次 miss + 49 次 hit）
- 单局成本：$0.30 → $0.09（**70% ↓**）
- 首 token 延迟：3-5s → 1-2s（**50% ↓**）

**重要修复**：`_inject_skill_directive()` 函数需要兼容 `msg.content` 是 list 的格式（带 cache_control 的结构）。

## 四、并发支持架构（v1.6+ 新增）

### 4.1 5 层并发设计

| Layer | 组件 | 文件 | 作用 |
|---|---|---|---|
| Layer 1: 进程级 | gunicorn / uwsgi（生产） | — | 多进程并行，无 GIL 限制 |
| Layer 2: 线程级 | ThreadingHTTPServer | `web_server.py` | 每请求一个线程 |
| Layer 3: 会话级 | SessionPool + SessionLock (RLock) | `concurrency.py` | 防同会话并发写入 |
| Layer 4: LLM 级 | LLMThrottle (Semaphore) | `concurrency.py` | 限制同时调 LLM 数量 |
| Layer 5: 持久化级 | AsyncSaveQueue + 文件锁 | `concurrency.py` | 存档异步写入 |

### 4.2 SessionPool（v1.6+ 新增）

```python
class SessionPool:
    """会话池 + LRU 淘汰 + 全局锁"""
    def __init__(self, max_sessions=50, session_ttl_seconds=3600):
        self._pool: OrderedDict[str, tuple[GameLoop, SessionLock]] = OrderedDict()
        self._pool_lock = threading.Lock()
        # ...
```

关键 API：
- `add(session_id, game)`：线程安全添加，触发 LRU 淘汰
- `get(session_id)`：线程安全获取，更新 LRU
- `cleanup_expired()`：清理过期会话（每 5 分钟后台线程调用）

### 4.3 LLMThrottle（v1.6+ 新增）

```python
class LLMThrottle:
    """LLM 调用限流（Semaphore）"""
    def __init__(self, max_concurrent=3, queue_timeout=120.0):
        self._semaphore = threading.Semaphore(max_concurrent)
```

关键设计：
- `max_concurrent=3`：同时最多 3 个 LLM 调用
- `queue_timeout=120s`：等待许可超时则抛 TimeoutError
- `with LLM_THROTTLE:` 上下文管理器自动获取/释放

## 五、KV 缓存架构（v1.6+ 新增）

### 5.1 缓存策略（3 层）

```
Layer 1: System Prompt Cache（永久，命中 100%）
  整个 _build_system_prompt() 输出（~1500 tokens）
  TTL: 5 分钟 ephemeral

Layer 2: 8 SKILL Directives（每回合变，不缓存）
  skill_directive 注入到 system prompt 末尾
  ~500 tokens × 50 回合 → 仍要传（每次变）

Layer 3: 对话历史（每次增长，不缓存）
  recent 3 回合 narrative
  增长 50-100 tokens/回合
```

## 八、内容划界：铁律层 / 可然层 / 自由层（v1.0 已完整实现）

### 8.1 三层定义

| 层级 | 定义 | 执行方式 |
|---|---|---|
| **铁律层** | 史实记载明确、无争议的事件和人物 | 代码级硬约束（SKILL-7 + iron_laws + post_validate） |
| **可然层** | 史书未记载但那个时代"可能发生"的事 | System Prompt 可然性原则 + 知识库 + SKILL-7 |
| **自由层** | 玩家的个人选择、感受、价值观 | 完全自由，规则引擎只计算后果 |

### 8.2 可然性检验四条

1. **时代一致**：不出现时代错位的器物、制度、观念
2. **身份一致**：小人物的行动不超出社会身份的合理范围
3. **逻辑一致**：虚构不与已知的史实矛盾
4. **因果一致**：虚构的因果链不改变已知的史实结果

## 九、行动点系统（v1.3+ 关键设计）

**这是游戏的核心节奏机制**：
- 每月固定 3-4 个行动点（按身份差异化）
- 行动点耗尽才跳月（不是固定 1 回合=1 月）
- 玩家可以"过日子"——织布、卖丝、纳粮、交税，一件件来

**time_cost 判定规则**：
| time_cost | 含义 | 例子 |
|---|---|---|
| 0 | 问询/观察（不消耗行动点） | "我看看窗外" |
| 1 | 半日功夫 | "我去茶馆听消息" |
| 2 | 一日功夫 | "我去苏州城里一趟" |
| 3 | 数日功夫 | "我做完一整批上供的丝绸" |

**按身份差异化**：
| 身份 | 行动点/月 |
|---|---|
| 织户（weaving_*） | 3 |
| 商人（merchant_*） | 4（应酬多） |
| 读书人（scholar_*） | 2（读书思考慢） |
