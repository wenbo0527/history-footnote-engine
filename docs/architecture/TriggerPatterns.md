# 触发模式库（27 个明清小说情节模式）

> 🆕 v1.7.30 整理
> 来源：[明清小说触发事件调研](../../eras/万历十五年/new_story/明清小说触发事件调研：可复用于历史注脚体验引擎的情节模式.md)
> 目的：把 27 个**具体触发模式**结构化，DM 可直接索引使用
>
> 注：调研文档原说"24 个"，但实际 7 大类 27 个（4+4+4+3+4+4+4）

## 设计原则

| 原则 | 含义 |
|---|---|
| **触发 ≠ 强制** | 事件出现后玩家可选应对方式，不是"必须走这条线" |
| **物象优先** | 一件东西引发连锁反应，比"突然发生一件事"更自然 |
| **因果链** | 玩家选择有后果（但后果可延迟）|
| **轻量触发** | 不是每个触发事件都要变成大剧情，有些只是"茶馆里听到一个消息" |
| **城市差异** | 苏州多商业陷阱，南京多权力博弈，杭州多宗教机缘，松江多产业转换 |

## 7 个新 EventId 命名空间

在现有 `fin.*` / `city.*` / `fam.*` / `gen.*` / `prop.*` / `inv.*` 之外，新增：

| 命名空间 | 含义 | 示例 |
|---|---|---|
| `trv.*` | 旅途意外 | `trv.ship_stuck` / `trv.robbed` |
| `comm.*` | 商业陷阱 | `comm.broker_lowball` / `comm.usury` |
| `gov.*` | 官府权力 | `gov.weaving_tax` / `gov.bribe_official` |
| `obj.*` | 物象触发 | `obj.token_exposed` / `obj.gold_bracelet` |
| `relig.*` | 宗教超自然 | `relig.nun_trap` / `relig.omen` |
| `reln.*` | 人际网络 | `reln.broker_wife` / `reln.guild_hall` |
| `dis.*` | 灾祸天命 | `dis.plague` / `dis.fire` / `dis.little_ice_age` |

## DM 决策树

每个触发模式 = 一个 DM 决策树入口。**核心循环**：

```
玩家行动（输入）
  ↓
DM 检查：是否触发某个 pattern？
  ├─ 是 → 调 pattern DM 素材（来自本文档/era.json triggers）
  │        ↓
  │     玩家选择应对方式
  │        ↓
  │     应用 fin.* / city.* / fam.* 等事件
  │
  └─ 否 → 普通 narrative
```
