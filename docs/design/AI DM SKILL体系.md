## 场控层

### SKILL-1 读场判断

**定义**：感知玩家当前状态，为后续所有决策提供依据。

**来源**：DM节奏控制设计（剧本杀DM控场核心能力）

**具体能力**：

| 判断维度 | 判断内容 | 输出 |
|---------|---------|------|
| 投入度 | 玩家输入长度、连续行为模式 | 高投入/低投入/陷入重复 |
| 情绪状态 | 玩家语言中的情绪信号 | 兴奋/焦虑/无聊/困惑/投入 |
| 叙事张力 | 当前场景有无冲突 | 高张力/中张力/低张力 |
| 进度位置 | 距离下一个史实锚点多久 | 即将触发/还有余裕/已过 |
| 路线倾向 | 玩家行为指向哪条路线 | 经营/科举/离乡/出家/抗税/… |
| 偏离程度 | 玩家是否跑偏/停滞 | 正常/轻微偏离/严重偏离/停滞 |

**实现方式**：
- 每回合结束前，DM在内部执行一次读场判断（不输出给玩家）
- 判断结果作为下一回合所有决策的输入
- 可通过Tool `get_state()` 获取 `player_idle_rounds` 等量化指标辅助判断

**读场信号清单**：

```
玩家高投入信号：
  - 输入 > 50字，包含具体行动和理由
  - 主动追问NPC细节
  - 对某个话题反复提及

玩家低投入信号：
  - 输入 < 10字，模糊指令
  - 连续3回合做类似的事
  - 不回应NPC的暗示

玩家困惑信号：
  - 反复问同一个问题
  - 输入中包含"不知道""怎么办"
  - 行动方向频繁切换

玩家跑偏信号：
  - 行动超出身份边界（如"我去找皇帝"）
  - 长时间停留在与主线无关的事上
  - 完全忽略已投放的线索
```

### SKILL-2 节奏控制

**定义**：根据读场判断结果，选择合适的时间模式推进叙事。

**来源**：DND四种时间模式（Robin D. Laws）+ 剧本杀DM控场

**四种时间模式**：

| 模式 | 触发条件 | DM行为 | 回合时长 |
|------|---------|--------|---------|
| **慢时间** | 重大抉择/关键NPC/史实锚点 | 逐句描写+内心独白+价值观发声 | 1回合=1刻钟~1时辰 |
| **现在时间** | 日常互动/NPC对话/市场交易 | 正常回应+环境描写 | 1回合=半天 |
| **抽象时间** | 重复行为/无事件日常/玩家说“等几天” | 一句话跳过+只汇报变化 | 1回合=数天~数月 |
| **锐切** | 史实锚点触发/需制造转折/玩家停滞 | 无过渡直接切入+制造紧迫感 | 瞬间切换 |

**切换决策树**：

```
读场结果 → 节奏决策

玩家面临重大抉择 → 慢时间
  └─ 插入价值观发声（SKILL-5）
  └─ 拆分为2-3回合展开

玩家正常互动 → 现在时间
  └─ 检查是否有线索可投放（SKILL-3）
  └─ 检查史实锚点是否临近（SKILL-4）

玩家重复/无所事事 → 抽象时间
  └─ 跳过空白时间
  └─ 检查是否需要"扶正"
  └─ 投放推动型线索（SKILL-3）

史实锚点触发 → 锐切
  └─ 无过渡切入
  └─ 史实锚定（SKILL-4）

玩家停滞/跑偏 → 扶正
  └─ NPC主动上门
  └─ 投放引导型线索（SKILL-3）
  └─ 或锐切拉回
```

**扶正机制**（来自剧本杀DM）：

| 玩家状态 | 扶正方式 | 例子 |
|---------|---------|------|
| 陷入重复循环 | 抽象时间跳过+新事件 | “又过了三日，镇上出了件大事” |
| 完全没有方向 | NPC主动找上门 | “隔壁王婶来敲门，说有急事” |
| 错过关键信息 | 补发线索 | “你突然想起之前听说的那件事” |
| 走太远 | 锐切拉回 | “你正要出门，却被一个消息拦住了” |

### SKILL-3 线索投放

**定义**：在恰当时机、以恰当方式释放信息，推动叙事和玩家探索。

**来源**：剧本杀DM线索发放技巧 + DM引导者行为模式

**线索四类型**：

| 类型 | 投放时机 | 投放方式 | 例子 |
|------|---------|---------|------|
| **推动型** | 玩家无所事事/陷入重复 | NPC闲话/环境变化 | “你听说镇上来了个新牙人” |
| **引导型** | 玩家跑偏/错过路线 | NPC主动提及/巧合事件 | “隔壁王婶说，最近科举要提前了” |
| **揭示型** | 玩家主动探索 | 搜索发现/NPC深谈 | “你翻到一本旧账簿，上面记录了……” |
| **压力型** | 需要制造紧迫感 | 传闻/倒计时 | “税吏三日后到镇” |

**投放原则**（来自剧本杀DM实践）：
- 给线索卡，不给答案——提供逻辑支点，让玩家自己推理
- 分层投放：表面→深层→关键，不一次给完
- 蝴蝶效应：用微小细节推动叙事，不用大事件砸人
- 投放时机根据玩家状态，不根据固定时间点

**与Tool的配合**：
- `check_rules(action， "insights")` → 查询可解锁的认知节点
- `query_knowledge(keywords， scene)` → 查询相关知识条目作为线索素材
- `save_event(consequences={"insight_unlocked": "ins_xxx"})` → 记录线索已投放

## 叙事层

### SKILL-4 史实锚定

**定义**：确保历史事件按史实发生，但呈现方式和玩家应对完全开放。

**来源**：剧情推进设计（史实锚点）+ 史实约束机制 + DM引导者行为模式（查证史实）

**三层操作**：

| 阶段 | DM行为 | Tool配合 |
|------|--------|---------|
| **铺垫** | 史实锚点临近时，通过传闻/环境变化/NPC闲话提前渗透 | `check_rules("forced_events")` 查询即将触发的强制事件 |
| **触发** | 锚点日期到达时，以自然场景呈现事件（通常用锐切） | `check_rules("forced_events")` 确认必须触发 |
| **应对** | 事件发生后，提供多种应对方式，但不改变事件本身 | `query_knowledge()` 查询相关知识支撑应对选项 |

**史实校验流程**（每回合叙事输出前）：

```
1. get_state() → 获取当前日期、已触发事件
2. check_rules("forced_events") → 是否有必须触发的史实事件
3. check_rules("action_boundary") → 玩家行动是否越界
4. query_knowledge() → 叙事涉及的历史细节是否准确
5. 生成叙事 → 确保与史实无矛盾
```

**核心原则**：
- 事件必须发生，但玩家如何应对完全自由
- 用叙事传递约束，不用系统弹窗
- 小人物改变不了历史大势，但可以选择自己成为什么样的人

---

### SKILL-5 价值观发声

**定义**：以内心独白形式呈现玩家内化的价值观，让选择有重量感。

**来源**：Disco Elysium“技能即性格” + 剧情推进设计（价值观发声）

**机制**：

```
era.json中定义的5个价值维度：
  - tradition_vs_change（守旧vs求变）
  - duty_vs_freedom（责任vs自由）
  - pragmatism_vs_idealism（务实vs理想）
  - independence_vs_network（独立vs依附）
  - acceptance_vs_resistance（顺从vs抗争）

每个维度有等级（1-5），等级越高，该价值观的"声音"越响亮。
```

**发声规则**：

| 价值观等级 | 发声方式 | 例子 |
|-----------|---------|------|
| 1-2级 | 偶尔闪过 | “你隐约觉得哪里不对” |
| 3级 | 明确内心独白 | “你想起父亲说过，读书人才有出路” |
| 4-5级 | 强烈发声，几乎成为选项 | “一个声音在说：这税不该交！但你又想起沈氏的话……” |

**触发时机**：
- 玩家面临与该价值观相关的抉择时
- 慢时间模式下，作为节奏放慢的叙事手段
- 不在现在时间/抽象时间中主动发声（避免干扰日常节奏）

**与Disco Elysium的对应**：
- DE的24个技能 = 我们的5个价值维度
- DE的技能等级 = 我们的价值维度等级
- DE的技能主动提供选项 = 我们的价值观以内心独白“说话”
- DE的技能等级越高声音越响 = 我们的价值等级越高发声越强

### SKILL-6 失败叙事化

**定义**：玩家行动失败时，不输出“你失败了”，而是将失败转化为新的叙事转折。

**来源**：Disco Elysium“失败也是故事” + 剧情推进设计

**转化规则**：

| 失败类型 | 叙事化方式 | 例子 |
|---------|-----------|------|
| 行动失败（技能不足） | 失败开启新路径 | 织不出花绫→但素缎卖给了另一个客商 |
| 说服失败 | NPC拒绝但暴露信息 | 王掌柜不降价→但透露了丝价下跌的原因 |
| 探索失败 | 看到意料之外的东西 | 找不到账簿→但翻到了一封信 |
| 选择失败（后果不如预期） | 后果比预期更复杂 | 交了税→但发现邻居没交，引发矛盾 |

**核心原则**：
- 失败不是终点，是岔路口
- 每个失败映射2-3种转折可能性
- 失败让故事更丰富，不是更贫乏
- 不存在“Game Over”，只有“故事转向”

## 约束层

### SKILL-7 三层裁判

**定义**：判断玩家行为和叙事内容落在铁律/可然/自由哪一层，执行对应规则。

**来源**：三层分类法 + 史实约束机制

**三层判断标准**：

| 层级 | 判断标准 | DM行为 | 例子 |
|------|---------|--------|------|
| **铁律层** | 史实记载明确、无争议 | 严格执行，不可变更 | 戚继光必须死、税必须交、矿监必须来 |
| **可然层** | 史实无记载但时代可能发生 | 可然性检验（时代/身份/逻辑/因果一致） | 丝织户认识一个秀才→可然；丝织户见到皇帝→不可然 |
| **自由层** | 玩家个人选择和感受 | 完全尊重，只计算后果 | 选择交税还是抗税、选择留还是走 |

**裁判流程**：

```
玩家行动 → 判断层级

铁律层？
  ├─ 是 → 执行史实，用叙事呈现约束
  └─ 否 → 可然层？
              ├─ 是 → 可然性检验
              │        ├─ 通过 → 生成叙事
              │        └─ 不通过 → 用叙事拒绝（不用系统弹窗）
              └─ 否 → 自由层 → 尊重选择，计算后果
```

**叙事化约束**（关键原则）：
- 铁律约束用叙事传递，不用系统弹窗
- “你改变不了皇帝的决定”→ 不写“系统提示：不可操作”，写“你托人递的话，石沉大海”
- 可然层拒绝用时代逻辑解释→ 不写“操作不允许”，写“守门人拦住了你：'你是什么人？'”

### SKILL-8 认知框架锁定

**定义**：玩家做出重大选择后，自动切换叙事的信息过滤方式，模拟“选择改变视角”。

**来源**：Disco Elysium思想内阁 + 剧情推进设计（认知框架锁定）

**机制**：

```
玩家选择路线 → 锁定认知框架 → 叙事信息过滤改变

例子：
  玩家选择"科举路线" → 锁定"读书人认知框架"
    → 叙事中自然突出：科举消息、文人圈动态、经义讨论
    → 叙事中自然抑制：丝价行情、牙行八卦、织工传闻

  玩家选择"经营路线" → 锁定"商人认知框架"
    → 叙事中自然突出：丝价波动、客商动向、行会消息
    → 叙事中自然抑制：科举消息、文人雅集、经义讨论
```

**与Disco Elysium的对应**：
- DE的思想内阁 = 我们的认知框架
- DE的思维方式解锁后永久改变角色 = 我们的路线选择后永久改变信息过滤
- DE的思维方式是双刃剑 = 我们的认知框架锁定后既突出也抑制

**实现方式**：
- era.json的 `insight_tree` 中定义每个认知框架对应的信息范围
- DM在叙事时，根据玩家当前认知框架，优先呈现框架内的信息
- 不是“看不到”其他信息，而是“不自然地忽略”——就像现实中你不会注意到与你无关的事

## SKILL之间的协作关系

```
每回合执行流程：

1. 读场判断（SKILL-1）
   ↓ 输出：玩家状态+叙事张力+进度位置

2. 节奏控制（SKILL-2）
   ↓ 根据读场结果选择时间模式
   ↓ 同时决定是否需要扶正

3. 线索投放（SKILL-3）
   ↓ 根据节奏和玩家状态投放线索

4. 史实锚定（SKILL-4）
   ↓ 检查是否有必须触发的史实事件

5. 三层裁判（SKILL-7）
   ↓ 判断玩家行动落在哪一层

6. 认知框架锁定（SKILL-8）
   ↓ 根据当前框架过滤信息

7. 价值观发声（SKILL-5）
   ↓ 仅在慢时间+相关抉择时触发

8. 生成叙事
   ↓ 如果涉及失败→失败叙事化（SKILL-6）

9. save_event() 记录本回合结果
```
