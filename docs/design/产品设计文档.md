## 二、架构设计（v3.1 增强版）

### 2.1 核心架构：单 Agent + 8 SKILL + 5 层并发 + KV 缓存 + 性能优化（v1.6.2）

```
┌─────────────────────────────────────────────────────────────────────┐
│                  Web Server (HTTP API)                              │
│   Layer 1: 进程级（gunicorn 多进程，扩展）                              │
│   Layer 2: 线程级（ThreadingHTTPServer + ThreadPoolExecutor）         │
│   🆕 A6: HTTP keep-alive（HTTP/1.1 + 5s timeout）                    │
│   🆕 A7: GZIP 压缩（Accept-Encoding → Content-Encoding）              │
│   🆕 A8: Cache-Control（HTML 5min, JSON no-store）                   │
│   🆕 D6: 请求限流（每 IP 60 req/min，LLM 端点 20 req/min）             │
│   🆕 /metrics 监控端点                                                │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│              DM Agent（唯一的LLM调用点）                                │
│   System Prompt = DM人格 + 时代配置 + SKILL Directives                │
│   13 个 Tool：规则引擎 + 记忆 + 知识库 + dice + skill_orchestration  │
│   🆕 A2: LLM Provider 全局缓存（避免每回合新建 ChatAnthropic）         │
│   🆕 A4: LangGraph graph 复用（state machine 一次构建）              │
│   🆕 B1: SKILL 选择性注入（按 intent_type 选 2-4 个 SKILL）           │
│   🆕 B3: Tool schema + system prompt 双重 cache_control              │
│   🆕 B6: 1h TTL 缓存（适合长会话）                                    │
│   🆕 D1: Tool 结果缓存（query_knowledge LRU + TTL）                  │
└──────────┬──────────────────────────────────────────────────────────┘
           │ Function Calling + KV Cache + LLM_THROTTLE
    ┌──────┼──────────┬──────────────┬────────────┐
    ▼      ▼          ▼              ▼            ▼
┌────────┐┌────────┐┌──────────┐┌─────────┐┌─────────────┐
│规则引擎 ││记忆管理  ││知识库检索  ││Dice掷骰  ││8 SKILL 编排  │
└────────┘└────────┘└──────────┘└─────────┘└─────────────┘
   ↑                                    ↑
   🆕 A1: era_config 全局缓存            🆕 skill_selector 选择器
```

**Layer 3（会话级）**：SessionPool + SessionLock（RLock）

**Layer 4（LLM 级）**：LLMThrottle Semaphore（max_concurrent=3）

**Layer 5（持久化级）**：AsyncSaveQueue + 文件锁

## 三、模块设计（同 v3.0）

### 3.1 DM Agent（v3.1 增强版）

#### 3.1.1 5 阶段行为模型（v3.1 同 v3.0）

阶段 0 → 阶段 1 → 阶段 2 → 阶段 3（后校验 + 重试） → 阶段 4

#### 3.1.2 KV 缓存集成（v3.1 增强）

```python
# system_prompt 加 cache_control（v3.1 默认 1h）
SYSTEM_PROMPT_CACHE = SystemPromptCache(ttl="1h")

# run + regenerate 都加 cache_control
system_message = SystemMessage(
    content=[{
        "type": "text",
        "text": system_prompt,
        "cache_control": {"type": "1h"},  # 1 小时 TTL
    }]
)
```

#### 3.1.3 LangGraph Graph 复用（🆕 v3.1 重大优化）

```python
# 旧：每回合重建 graph
self.graph = self._build_graph(state_ref=state_ref)  # 50-300ms

# 新：复用 graph + 只更新 state_ref
if hasattr(self, '_graph_compiled') and self._graph_compiled is not None:
    # 通过 _state_ref_slot_ref[0] = state_ref 切换数据
    self.graph = self._graph_compiled
else:
    self.graph = self._build_graph(state_ref=state_ref)
    self._graph_compiled = self.graph
```

**收益**：50 回合省 **~10-15s CPU**（每回合 200-300ms → <1ms）

#### 3.1.4 SKILL 选择性注入（🆕 v3.1 重大优化）

不是所有回合都需要 8 个 SKILL 全部触发。

```python
from history_footnote.skill_selector import select_skills, filter_skill_directive

# 按 intent_type 选 SKILL
selected_skills = select_skills(intent_type, state=state_ref)
# 例如：inquire → ["skill_1", "skill_7"]（仅 2 个）

# 过滤完整 directive 文本
filtered_directive = filter_skill_directive(full_directive, selected_skills)
```

**节省估算**（单回合 500 tokens）：
- action：500 → 250 tokens（50% ↓）
- inquire：500 → 125 tokens（75% ↓）
- describe：500 → 190 tokens（62% ↓）

### 5.2 v3.1 新增：Tool 结果缓存（D1）

```python
from history_footnote.web_enhancements import TOOL_RESULT_CACHE

@TOOL_RESULT_CACHE.cached("query_knowledge")
def query_knowledge(keywords, scene, entry_ids):
    return knowledge_base.query(...)
```

**收益**：同一关键词重复查询 → **1000x 加速**（100ms → 0.1ms）

## 十、SSE Streaming 输出（🆕 v3.1 重大 UX 改进）

### 10.1 新增端点：`/api/input_stream`

```
POST /api/input_stream
Content-Type: application/json
{
  "session_id": "...",
  "input": "..."
}

→ SSE Response (text/event-stream):
event: thinking
data: DM 正在分析场景...

event: thinking
data: DM 正在生成叙事...

event: chunk
data: 灶房里，

event: chunk
data: 沈氏蹲在灶台前

event: chunk
data: 吹火，呛得直咳嗽。

event: done
data: {"voice_options": [...], "intent_type": "action", "validation_passed": true}
```

### 10.2 前端使用示例

```javascript
const evtSource = new EventSource('/api/input_stream?session_id=xxx&input=xxx');
let buffer = '';

evtSource.addEventListener('thinking', (e) => {
    showStatus(e.data);  // 显示 "DM 在思考中..."
});

evtSource.addEventListener('chunk', (e) => {
    buffer += e.data;  // 逐字显示
    updateNarrative(buffer);
});

evtSource.addEventListener('done', (e) => {
    const data = JSON.parse(e.data);
    showVoiceOptions(data.voice_options);
    evtSource.close();
});
```
