### 2.2 图标生成策略

**用 mmx image 生成**：
```bash
# 5 个 stat 图标
mmx image "国风水墨风格，宋代银锭图标，简洁笔画，浅米色背景，半透明" --aspect-ratio 1:1 --out icons/stats/cash.svg
mmx image "国风水墨风格，明代织机图标，简洁笔画，浅米色背景" --aspect-ratio 1:1 --out icons/stats/loom.svg
mmx image "国风水墨风格，明代官印图标，朱砂色，简洁笔画" --aspect-ratio 1:1 --out icons/stats/reputation.svg
mmx image "国风水墨风格，红色小点，简洁笔触，类似红叶" --aspect-ratio 1:1 --out icons/stats/action.svg
mmx image "国风水墨风格，桃符护身符，红色，简洁笔触" --aspect-ratio 1:1 --out icons/stats/health.svg

# 6 个 nav 图标
mmx image "国风水墨卷轴图标，褐色，简洁笔画" --aspect-ratio 1:1 --out icons/nav/archive.svg
mmx image "国风水墨方志书册图标，褐色" --aspect-ratio 1:1 --out icons/nav/wiki.svg
mmx image "国风水墨玉佩图标，碧色，简洁" --aspect-ratio 1:1 --out icons/nav/choice.svg
mmx image "国风水墨古书图标，褐色" --aspect-ratio 1:1 --out icons/nav/recap.svg
mmx image "国风水墨朱砂印章图标，红色" --aspect-ratio 1:1 --out icons/nav/settings.svg
mmx image "国风水墨红灯笼图标，红色，简洁" --aspect-ratio 1:1 --out icons/nav/home.svg
```

**注意**：mmx 输出是 PNG 格式（不是 SVG），需要：
- 选项 A：直接用 PNG（图片图标）—— 简单
- 选项 B：用 Inkscape / 手动转 SVG —— 复杂

**建议先用 PNG**（简化为 webp 节省体积），效果一样。

### 2.3 角色插图生成

**Prompt 模板**（每个 identity + gender 一张）：

### 2.4 集成到代码

**A. 替换 emoji → SVG（以 AppHeader 为例）**

```svelte
<!-- 旧 -->
<span class="app-header-stat" title="现金">💰 {stats.cash.toFixed(2)}</span>

<!-- 新 -->
<img src="/icons/stats/cash.webp" alt="现金" class="stat-icon" />
<span class="app-header-stat">{stats.cash.toFixed(2)}</span>
```

**B. 替换 👤 → 角色插图（CharCard）**

```svelte
<!-- 旧 -->
<div class="char-card-avatar" aria-hidden="true">👤</div>

<!-- 新 -->
<img
  src="/character/{character.identity}_{character.gender}.webp"
  alt={character.name}
  class="char-card-portrait"
/>
```

**C. CharCard 动态加载**

```svelte
<script lang="ts">
  let portraitUrl = $derived(`/character/${character.identity}_${character.gender}.webp`);
  let fallbackEmoji = $derived('👤');  // 图片加载失败时降级
</script>

<img
  src={portraitUrl}
  alt={character.name}
  class="char-card-portrait"
  onerror={(e) => (e.currentTarget as HTMLImageElement).style.display = 'none'}
/>
```

| 11 个图标 | 1 张 | ×3 候选 | 33 张 |

| 类型 | 单张大小 | 数量 | 总计 |

| **总计** | - | - | **~3.6MB** |

**接受**（小于一个 React 包大小）。

## 5. 风险 + 缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| mmx 图像风格不统一 | 中 | 视觉混乱 | 严格 prompt 模板（"国画风格，浅米色背景"） |
| 角色插图脸不像玩家期望 | 高 | 体验下降 | 多生成（--n 3）+ 人工选最佳 |
| 图片文件大，拖慢加载 | 中 | 体验差 | 转 webp（压缩 80%）+ 懒加载 |
| 跨浏览器 webp 兼容 | 低 | 体验差 | webp 现代浏览器全支持（95%+） |
| mmx 服务不稳 | 中 | 生成失败 | 加 try/except fallback 到 emoji |
| mmx image 输出不是 SVG | 高 | 不能缩放 | 用 PNG/webp（接受），或在 Inkscape 转 |

---

## 6. 性能影响

| 维度 | 当前（emoji） | 优化后（webp） | 影响 |
|---|---|---|---|
| 视觉一致性 | ❌ 杂乱 | ✅ 国风水墨 | 巨幅提升 |
| 加载大小 | 0（emoji 字体） | ~3.6MB（首次） | 一次性 |
| 加载时间 | 0ms | 200-500ms（首次） | 可接受 |
| 缓存 | 浏览器字体 | 浏览器 disk cache | 二次 0ms |
| 跨设备一致性 | ❌ 系统 emoji 各异 | ✅ webp 全一致 | 巨幅提升 |

## 9. 决策点

**需要用户决定**：

1. **先做图标还是先做插图？**
   - 图标：11 张，工作量小，立刻提升 UI
   - 插图：12 张 + 风格选定，工作量大，体验提升更高

2. **是否改 brand 整体风格？**
   - 现在：emoji + 文字密集
   - 选项 A：保留 emoji + 加水墨点缀（最低风险）
   - 选项 B：完全替换 emoji 为 webp 图标（高一致性）
   - 选项 C：每页加插图背景（高沉浸感）

3. **mmx OAuth 是否现在就做？**
   - 需要在终端浏览器授权（30 秒）
   - 一旦完成，mmx 所有能力可用

**我的推荐**：先完成 OAuth → 生成 11 个图标（半天）→ 体验改善 → 再考虑插图。
