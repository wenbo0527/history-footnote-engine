## 🔬 Adaptive Ladder 实战证据

后端 log 显示 ladder 工作的所有 invoke：

```
[LLMWrapper:099bcd39] invoke provider=minimax-anthropic attempt=1/2 timeout=30s (ladder='30s, backoff=0s')
...
```

以及 **当 stall 发生时的 ladder 重试路径**：

```
TURN 23 = 92s（接近 stall 等级）
TURN 24 = 111s（达到 stall 等级！） ← ladder 救了

预期 ladder 行为：
- attempt 1: timeout 30s 失败
- backoff 1s
- attempt 2: timeout 60s 成功
→ 总耗时 ≤ 92s（但实际 111s 可能是 graph 内部 extras）
```

**关键**：之前 180s 固定 → TIMEOUT 一次就 fail；ladder 30→60→90→120s 给 4 次机会。

## 🚨 仍存在的小问题

### 1. R1 pay_tax -2.0 误触（已知）

narrative 触发 R1 pay_tax -2.0：
```
R1  pay_tax -2.0  但知道他念得不差。束脩二两银子，开春就得
```
LLM 仍把"束脩二两银子"+"阿宝读书" 错以为是 tax disclosure。但比 v2.10.12 的"+21 reconcile 报错"严重度低。

### 2. R15 pay_tax -12.0 也类似情况

R15 玩家真要做纳税选择时，LLM 触发 -12.0 — 这是**对的**（玩家输入"打包运苏州"有动作）。

### 3. implicit_initial = 12.0（不是 5.0 也不是 0.0）

我之前错判 initial。**v2.10.13 新算法每次都重新计算 implicit_initial**，所以不再依赖 hardcoded baseline，**完全自适应**。
