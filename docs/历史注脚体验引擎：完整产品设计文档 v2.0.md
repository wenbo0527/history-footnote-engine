## 一、产品定义

### 1.1 一句话定义

AI当DM，你当历史里的小人物。通过对话体验一个历史时期，历史走势不可改，但你的选择路径完全开放。

### 1.2 不是什么

- 不是"历史课"——没有知识点考核，只有沉浸体验
- 不是"架空冒险"——史实是铁律，不能改写
- 不是"经营模拟"——核心不是数值优化，是人在时代中的处境
- 不是"多Agent协作系统"——单Agent+Tool，最小架构
- 不是"游戏关卡"——**没有Game Over**，只有"故事转向"（v1.4+）

### 1.3 是什么

**配置驱动的AI叙事系统 + Disco Elysium风格认知驱动系统**（v1.4+）。

核心循环：
```
玩家输入 → 8 SKILL编排 → DM生成叙事 → 规则引擎计算状态 → DM反馈结果 → 玩家输入
         ↑                    ↓
   （含 voice_options）  （含内在声音选项）
```

通用化的本质：把循环里每个环节的"硬编码"变成"可配置"。换一份配置表+一份知识库，就能换一个时代。

**v1.4+ 新增**：在 DE 风格启发下，**8 SKILL 公共技能系统**成为 DM 编排的核心，每回合先跑 SKILL，再调 DM 生成叙事。

### 1.4 产品定位

**"受约束的自由"**——历史事实是铁律，但玩家的选择路径完全开放。

约束不是限制，是体验的来源：正因为你知道历史不可改，小人物的挣扎才有了重量。

市面上无同类产品。AI Dungeon类无史实约束，DE 类无历史约束，Jomini Engine 是唯一学术先例（宏观战略视角），但无 AI DM 层。

## 三、模块设计

### 3.1 DM Agent（v1.4+ 三阶段行为模型）

#### 3.1.1 DM 不是被动响应器

当前设计把DM当成"玩家输入→调Tool→输出叙事"的被动函数，但真正的DM有三个主动行为是纯Function Calling覆盖不了的：

| 主动行为 | 场景 | 纯调Tool的问题 |
|---|---|---|
| **推进节奏** | 玩家犹豫、漫无目的、节奏拖沓 | 等玩家输入才响应，游戏陷入停滞 |
| **提示引导** | 玩家不知道该关注什么，认知解锁需要引导 | 等玩家碰巧提到关键词才触发，认知树形同虚设 |
| **校验史实** | 生成叙事前需要核对已触发事件和知识 | 等后校验来纠错，叙事已经矛盾了再修很生硬 |

**v1.4+ 关键决策**：把"DM 自主判断"改为"**8 SKILL 编排层确定性计算** + DM 创意生成"。确定性逻辑归代码，创意性判断归 DM。

#### 3.1.2 v1.4+ 四阶段行为模型（重写）

DM每回合**四阶段串行推理**：

```
阶段0：8 SKILL 编排（v1.4+ 新增，确定性逻辑）
  → 跑完 8 个 SKILL：读场判断 → 节奏控制 → 线索投放 → 史实锚定 →
                    价值观发声 → 失败叙事化 → 三层裁判 → 认知框架锁定
  → 输出：DMContext（含 pacing/lead/voices/cognitive_frame 等）+ skill_directive 文本

阶段1：态势评估（Reason about Situation）
  → DM读取玩家输入 + SKILL Directives + 当前状态
  → 自主判断：
    - 玩家是否在犹豫？→ 需要推进节奏（SKILL-2 已给建议）
    - 是否有认知节点接近解锁？→ 需要暗示引导（SKILL-3 已给线索）
    - 是否有历史锚点即将触发？→ 需要铺垫（SKILL-4 已铺垫）
    - 玩家行动是否可能触碰史实？→ 需要查证（SKILL-7 已给约束）
  → 调用Tool：get_state / recall_events / check_rules / query_knowledge

阶段2：叙事生成（Generate Narrative）
  → 基于 SKILL Directives + 阶段1 Tool结果，生成叙事
  → 必须输出 voice_options（2-4 个 DE 风格内在声音选项）
  → 调用Tool：check_rules（行动边界）/ query_knowledge（史实核对）
  → 输出：DMResponse（结构化 + voice_options + intent_type）

阶段3：状态确认（Confirm & Save）
  → 确认叙事中的状态变更与规则引擎一致
  → 保存事件到记忆 + voice_options 到 state（v1.5+ 持久化）
  → 调用Tool：save_event / get_state（二次确认）
```

**关键区别（v1.4+）**：不是"玩家输入→被动响应"，也不是"DM 自主判断节奏"——而是"玩家输入 → **确定性 SKILL 编排** → DM 创意生成 → 状态确认"。确定性逻辑归代码，DM 只决定"怎么讲"。

#### 3.1.3 主动行为的边界（v1.4+ 修订）

**核心问题**：DM 需要主动推进节奏，但推进的方向不能偏离历史主线。v1.4+ 用 **SKILL 编排层**解决这个问题：

| 层级 | 谁决定 | 决定什么 | 约束来源 |
|---|---|---|---|
| **方向层** | 8 SKILL 编排层（代码） | 推进的大方向（朝哪个历史锚点走、哪个认知节点该解锁、用哪个时间模式） | `pacing_anchors` + `voices` + `cognitive_frames` |
| **方式层** | DM | 用什么叙事手段推进（NPC对话、环境描写、事件触发） | 可然性四原则 |
| **校验层** | SKILL-7 三层裁判 | 推进结果是否违反铁律 | `iron_laws` + SKILL-7 |

**v1.4+ 的方向层注入示例**：
```
[SKILL-2 节奏控制 → slow_time]
  玩家面临重大抉择 → 慢时间，逐句展开
[SKILL-4 史实锚定]
  锚点: 春税征收（3回合后触发）
  铺垫线索: 镇上在传今年春税的数额
[SKILL-5 价值观发声]
  [中] 算盘声（强度3/5）：一个声音在你脑子里拨算盘...
[SKILL-8 认知框架锁定]
  → weaving
  突出呈现: 织机/经线/梭子
  抑制: 科举消息
```

#### 3.1.4 System Prompt 模板（v1.4+ 完整版）

```python
SYSTEM_PROMPT = """
你是{era_name}的历史DM。

## 时代背景
{era_background}

## 你的三重身份
你是叙事者、仲裁者、引导者。三者在同一回合内融合执行。
- 叙事者：用细节而非数字，用场景而非概括
- 仲裁者：严格执行规则引擎的计算结果，绝不篡改数值
- 引导者：当 SKILL Directives 指示需要推进节奏时，用叙事手段自然推进

## 每回合行为流程
1. 读取上下文：玩家输入 + 8 SKILL Directives（方向层）
2. 生成叙事：融合所有信息，生成叙事
3. 输出状态：报告变量变更、事件记录、价值观偏移
4. 输出 voice_options：2-4 个内在声音选项（DE 风格）

## 历史红线（不可违反）
{iron_laws_list}

## 小人物身份约束
你是{player_identity.role}，{player_identity.social_class}。
可接触：{action_boundaries.can_access}
不可接触：{action_boundaries.cannot_access}

## 可然性原则
{plausibility_rules}

## 时代核心变量
{variables_with_max_shift}

## 行动点（v1.3+ 关键约束）
每月固定 {action_points_max} 个行动点。time_cost 0=问询，1=半日，2=一日，3=数日。

## 输出格式（严格遵守）
```json
{
  "narrative": "具体场景描写（300-800字）",
  "is_action": true,
  "time_cost": 2,
  "intent_type": "action",  // action | inquire | describe | voice
  "voice_options": [
    {"voice_id": "voice_xxx", "voice_name": "内在声音名", "intent_text": "按这个声音做的事"}
  ],
  "state_changes": {"variable_id": +1.0},
  "events_to_save": ["事件摘要"],
  "updates": {"insight:ins_01": "unlocked"}
}
```
"""
```

**v1.5+ intent_type 语义**：
- `action`：真行动（消耗行动点）
- `inquire`：问询/观察（不消耗行动点）
- `describe`：玩家补充身份/环境/性格（不消耗行动点，但 DM 应承认）
- `voice`：玩家选择了某个内在声音选项（强制 is_action=true）

**v1.5+ voice_options 设计原则**：
- 每个叙事回合都应给出 2-4 个 voice_options（DE 风格的"脑海中的几个声音"）
- 选项要**性格鲜明**——同一件事，不同声音给不同建议
- 选项要**符合时代**——不出现现代思维
- 最后一个隐藏选项是**自由输入**

### 3.6 成长感系统（v1.4+ 实现状态：80%）

### 3.7 Dice Engine（v1.4+ 新增模块，**v1.0 文档未提**）

**DND 风格掷骰子系统**：

- 支持标准 DND 骰子表达式：`d4`, `d6`, `d8`, `d10`, `d12`, `d20`, `d100`
- 组合表达式：`2d6+3`, `1d20-1`, `3d8+2d6`
- 优势/劣势（双骰取高/取低）
- 大成功/大失败（d20=20/1）

**设计哲学**：DiceEngine 是**确定性服务的随机性补充**——
- 规则引擎：严格计算（赋税、变量变更）
- DiceEngine：随机判定（谈判、遭遇、机会）

**当前使用情况**：实现但**未接入** DM 主循环（v1.5+ 计划在玩家"问询/挑战"时触发）。

## 四、内容划界：铁律层 / 可然层 / 自由层（v1.0 已完整实现）

### 4.1 三层定义

| 层级 | 定义 | 执行方式 |
|---|---|---|
| **铁律层** | 史实记载明确、无争议的事件和人物 | 代码级硬约束（SKILL-7 + iron_laws） |
| **可然层** | 史书未记载但那个时代"可能发生"的事 | System Prompt 可然性原则 + 知识库 + SKILL-7 |
| **自由层** | 玩家的个人选择、感受、价值观 | 完全自由，规则引擎只计算后果 |

### 4.2 SKILL-7 三层裁判（v1.4+ 实现）

```python
def skill_7_three_layer_verdict(era_config, player_input, state) -> ThreeLayerVerdict:
    # 1. 铁律层：检查 iron_laws 中是否被玩家输入触发
    # 2. 可然层：检查 action_boundaries（不能访问/不能影响）
    # 3. 自由层：默认 allow
    return verdict
```

### 4.3 可然性检验四条

1. **时代一致**：不出现时代错位的器物、制度、观念
2. **身份一致**：小人物的行动不超出社会身份的合理范围
3. **逻辑一致**：虚构不与已知的史实矛盾
4. **因果一致**：虚构的因果链不改变已知的史实结果

### 4.4 体验产生于三层之间的张力

1. 铁律层施加约束 → "海瑞死了"
2. 自由层做出选择 → "我去不去祭拜？"
3. 可然层呈现后果 → "你去了，发现海瑞家连丧葬费都凑不齐"

## 六、行动点系统（v1.3+ 关键设计，v1.0 文档未提）

### 10.2 游戏主循环（v1.4+ 修订版）

```python
def _run_round(self, player_input: str) -> None:
    # === 步骤 0：输入预处理 ===
    player_input = self._preprocess_input(player_input)

    # === 步骤 1：8 SKILL 编排（v1.4+ 新增，确定性逻辑） ===
    intent_type = _detect_intent_type(player_input)  # 规则判定
    skill_directive = run_all_skills(...)  # 8 个 SKILL → system prompt 注入

    # === 步骤 2：规则引擎预计算 ===
    action_check = self.rule_engine.check_action(...)
    anchors = self.rule_engine.check_forced_events(...)
    triggers = self.rule_engine.check_triggers(...)
    pacing_directives = self.rule_engine.check_pacing(...)
    insight_candidates = self.rule_engine.check_insights(...)

    # === 步骤 3：记忆召回 ===
    recent_events = self.memory.get_recent(rounds=3)
    related_events = self.memory.recall_by_context(...)

    # === 步骤 4：DM Agent 生成叙事（一次API调用） ===
    # ⚠️ 当前没做 MAX_RETRY=2 重试和 post_validate
    dm_response = self.dm_agent.generate(
        player_input=player_input,
        current_state=...,
        action_check=action_check,
        anchors=anchors,
        skill_directive=skill_directive,  # v1.4+
        ...
    )

    # === 步骤 5：应用变量变更 ===
    adjust_result = self.rule_engine.apply_changes(dm_response.state_changes)

    # === 步骤 6：保存事件 + voice_options ===
    self.memory.save_events(dm_response.events_to_save)
    self.state.last_voice_options = dm_response.get("voice_options", [])  # v1.5+

    # === 步骤 7：呈现给玩家（voice_options 也展示） ===
    self._display_narrative(dm_response.narrative)
    self._display_voice_options(dm_response.voice_options)  # v1.5+
```
