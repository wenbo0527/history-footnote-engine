## 🔍 测试中发现的事实（不显于 mock 测试）

### ✅ 真实 LLM 工作链路完整

```
turn N → POST /api/input { session_id, input }
       → DM Agent LangGraph 8-12 个 LLM 调用
       → 真实 miniMax API（200 OK 200+s）
       → narrative + facts_extraction + 事件判定
       → state 推进（round / cash / variables）
```

每回合 backend `graph.invoke` 平均耗时 7-30s，最坏 60s+（真实 LLM 重试）。

### ⚠️ 真实 LLM 服务的 hiccup pattern

观察到 backend log：
```
graph.invoke: 14569ms (steady)
graph.invoke: 60797ms (LLM hang 60s 后回来)
graph.invoke: 74036ms (LLM hang 74s 后回来)
```

意味着 miniMax API **偶发 30-90s 不响应**，但**总会回到** — 此 bug 不是项目代码。

### 🐛 **真实测试发现的 bug — path 层数错（已修复）**

E2E 准备工作时发现（README.md 中 v2.10.11 路径修复之一）：

```python
# src/history_footnote/saves/resource_cache.py:27
- _ROOT = Path(__file__).resolve().parent.parent.parent
+ _ROOT = Path(__file__).resolve().parent.parent.parent.parent
# 注意：saves/resource_cache.py 需 4 层 .parent
# src/history_footnote/saves/resource_cache.py
# → src/history_footnote/saves/   (1)
# → src/history_footnote/        (2)
# → src/                         (3)
# → <project_root>               (4)
```

**症状**：`/api/start` 返回 500 `FileNotFoundError: Era package not found: wanli1587`（era 目录明明存在）。

**根因**：`save/` 子包移位到 `saves/` 后（v2.10.9 P0-1 重构），路径计算少了 1 层 `.parent`。

**修复**：4 层 → 5 层 .parent。

## ✅ 验证 / ❌ 失败 / ⚠️ 中性

### ✅ Successes

1. **真 LLM 调用链路 100% 工作**（当 LLM 服务响应时）
2. **状态持久化跨 resume** — session `wanli1587_20260719_181815` 跨 4 次 attempt 累计 turn 22 都保持一致
3. **financial state 推进** — 0 → 33.7 两（真实玩家可重现的"经营绸缎赚银子"路径）
4. **variables 推进** — silver_pressure / tax_burden / livelihood 等 9 维度都被 LLM 触发调整

### ⚠️ Caveats（部署建议 + 后续优化）

1. **30+回合运行 ~30 分钟** — 单回合 LLM 触发 6-12 次（narrative 重试 + facts_extraction + voice_options）
2. **真实 LLM 服务端可能 stall**（30s-120s）但**总会恢复**。**建议**：web_server 加 adaptive timeout（自动 retry）
3. **session server-side TTL** — 30+ 分钟会过期，影响 `--resume` 用例。**建议**：web_server 加 session persist to disk on idle

### ❌ 失败（已修）

1. **ERAS_DIR 路径 bug**（commit 准备中）— 上面已修
2. **post() HTTPError 处理漏 TimeoutError**（已修在测试脚本里）
3. **Session TTL**（未修）— 是 by-design，文档化清楚
