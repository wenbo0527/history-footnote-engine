## ✅ F3：Session TTL Sweep + 自动落盘

### 问题（最关键的）

之前 `cleanup_expired()` 只 remove，**不存档**。30 分钟 TTL 触发后：

```
session 被驱逐 → /api/state 返 404 → 玩家丢失所有 progress
```

之前 e2e 测试第 4 次 resume attempt 就因这个失败。

### 修复

[src/history_footnote/game/concurrency.py](file:///Users/mac/Documents/trae_projects/history_footnote/src/history_footnote/game/concurrency.py#L270-L358)

新增三层结构：

```python
# 1. _sweep_idle_sessions_once() — 实际清理逻辑
def _sweep_idle_sessions_once() -> int:
    """每 N 秒 daemon 调用一次：
    - 找出 last_used_at + ttl < now 的 session
    - 用 SaveManager.save_state() 落盘 auto slot
    - 从 SESSION_POOL remove
    """
    
# 2. _sweep_daemon() — 后台线程，60s 一次
def _sweep_daemon() -> None:
    while True:
        sleep(60)
        try:
            n = _sweep_idle_sessions_once()
        except Exception as e:
            logger.warning(...)

# 3. ensure_sweep_daemon_started() — 幂等启动器
def ensure_sweep_daemon_started():
    """模块导入即启动（web_server 启动时 daemon 也活）"""
    
ensure_sweep_daemon_started()  # 模块级
```
