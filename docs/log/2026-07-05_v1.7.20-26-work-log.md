### v1.7.22 — voice_options context-aware 兜底

**问题 1**：v1.7.21 兜底总是 3 个**完全相同**的 default 选项
- "谨慎行事/按本心/问问旁人" — 5 回合都一样

**问题 2**：2 个"自由输入"按钮
- 后端兜底注入 `voice_freetext` + 前端 hardcode = 重复

**根因**：

**解决方案**：

- 前端 `hasFreetext` 检测，跳过重复按钮

**新增 4 种时代声音**：
- 🔢 算盘声（银/钱/税）

- 🎨 手艺人的骄傲（织/布/丝）

**提交**：`eef795e v1.7.22: voice_options context-aware 兜底`

### v1.7.25 — 末尾问号 3 重保障

**用户反馈**：
> 请帮我检查，每次选择的时候需要有问题

**实测 v1.7.24 发现**：
- 5/5 narrative 末尾**无问号**
- v1.7.24 prompt 强化了但 LLM 不一定遵循

**3 重保障**：

**A. 后端 narrative_sanitizer 兜底**（最重要）
```python
def _ends_with_question(text: str) -> bool:
    tail = text[-30:].strip()
    return ("?" in tail) or ("？" in tail)

if not _ends_with_question(cleaned):
    fallback_questions = ["你想做什么？", "接下来怎么办？", "你怎么应对？"]
    cleaned += fallback_questions[hash(cleaned[-20:]) % 3]
```

**B. LLM 重试扩展**（`dm_agent.py`）
- 触发条件：`len < 100` OR `末尾无问号`
- 重试成功条件：长度 OK + 末尾问号 OK

**C. prompt 强化**（system_base.md）
- 新增 4 个 ❌ 反例（陈述句 / NPC 等待 / NPC 说话 / 元层反问）

**修复后实测**（5 回合）：
- 5/5 末尾都有问号 ✓
- 触发机制混合：2 LLM 重试 / 1 后端兜底 / 2 LLM 真实遵循

**提交**：`fde14ea v1.7.25: narrative 末尾问号兜底`

## 🔍 三、技术亮点

### 1. 多层兜底模式

任何关键 UX 保证都不能只靠单点：

```
LLM 层（prompt 引导）
   ↓ 不一定遵循
重试层（不符合则重试）
   ↓ 仍可能失败
后端层（强制追加）
   ↓ 服务降级
前端层（永远可用）
```

### 2. "困境驱动"叙事设计

参考 DE（Disco Elysium）模式：
- **场景描写**（150-200 字）：环境、氛围、NPC 状态
- **冲突呈现**（100-150 字）：显式标出当前困境——给数字、给压力
- **声音对峙**（80-120 字）：2-3 个内在声音的内心独白
- **明确邀请**（30-50 字）：**必须**以**问句**结尾，引导玩家决策

### 3. "概率性模型"心态

LLM 是概率性模型，**prompt ≠ 执行**：
- 必须有重试机制
- 必须有后端兜底
- 必须有前端最后防线

### 4. 自动化测试基础设施

- **单元测试** 24 个（每个模块独立）
- **E2E 测试** 13 phase 52 断言
- **字段一致性** 8 测试（防 API 漂移）
- **SSE 流式** 5 回合测试
- **OpenAPI 自动生成**（docs 与代码同步）

## 💎 六、核心经验

### 1. "永远有兜底"原则
- 后端兜底注入
- 前端永远显示"自由输入"
- 关键字段不依赖 LLM

### 2. 测试驱动是底线
每个新功能 = 新测试。100% 通过率是产品质量保障。

### 3. "显式优于隐式"
- 字段名规范（FIELD_REGISTRY）
- OpenAPI 自动生成
- 字段一致性测试

### 4. DE 风格核心
玩家需要**显式的困境**才能**做出选择**——空泛的"你打算怎么做"无意义。

### 5. 隐藏 bug 用直接调
`_format_state` 单元直接调才能发现 Python 缩进错误。HTTP 端到端测不出来。
