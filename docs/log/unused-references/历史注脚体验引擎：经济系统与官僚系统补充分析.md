# 历史注脚体验引擎：经济系统与官僚系统补充分析

## 一、核心结论

**需要补充，但不是做“模拟经营”的数值系统，而是做“叙事约束”的规则系统。**

现有era.json有9个变量（silver_pressure、livelihood、tax_burden等），但缺少两个关键层：
- **经济系统**：玩家做经济决策时，没有“多少钱算多、多少钱算少”的锚点，触发事件无法量化
- **官僚系统**：玩家遇到官府时，不知道“找谁、花多少、等多久”，触发事件没有规则

补充原则：**为叙事服务，不为模拟服务**。玩家不需要经营一家丝绸店的完整财务报表，但需要知道“一两银子能买什么”“打点一个书吏要多少钱”。

## 二、经济系统：需要补充什么

### 2.1 现有覆盖

era.json v1.1 已有：
- `silver_pressure`（白银紧缺压力）
- `livelihood`（生计稳定度）
- `tax_burden`（赋税负担）
- 5种丝织品种（湖丝、纺丝、缎、绢、纱）
- 5档行情周期

### 2.2 缺口：价格锚点体系

玩家需要知道“一两银子能干什么”，否则所有经济触发事件都是空谈。

**万历年间江南价格锚点**（基于《金瓶梅》物价和史料推算）：

| 品类 | 价格 | 来源 |
|------|------|------|
| **米** | 0.6-0.8两/石（丰年0.5，灾年2-3两） | 万历朝平均0.638两/石 |
| **一匹上等丝绸** | 3-5两 | 《金瓶梅》缎子铺日销500两，约100-150匹 |
| **一匹普通棉布** | 0.3-0.5两 | 松江布“三丈为匹”，换棉花或钱 |
| **织工日工钱** | 0.03-0.05两（30-50文） | “计日受值”，月入约1两 |
| **一张织机** | 3-5两 | 木织机，含配套工具 |
| **桑叶一担** | 0.03-0.1两（正常年景）；0.3两（缺叶年） | 施润泽故事中缺叶年暴涨 |
| **雇一个丫鬟** | 5-10两（买断） | 《金瓶梅》中丫鬟价格 |
| **县城小院** | 30-50两 | 盛泽镇上两间小房 |
| **知县年俸** | 27两（正七品法定） | 实际靠灰色收入维持 |

**关键换算感**：
- 一两银子 ≈ 一个织工一个月的工钱
- 一匹上等丝绸 ≈ 一个织工三到五个月的收入
- 一张织机 ≈ 一个织工半年的收入
- 西门庆行贿50两 ≈ 一个织工四年的收入

### 2.3 缺口：成本-收益模型

玩家经营丝织作坊，需要知道“织一匹绸能赚多少”：

**盛泽小户织一匹湖丝绸的成本**：
- 桑叶+蚕种：约0.5两（正常年景）
- 自己织（不计工钱）：0
- 交给牙行：牙行抽佣约10%
- 牙行收购价：约2-3两/匹
- **净收入**：约1.5-2两/匹（含自己劳动）
- **月产量**：小户1-2张织机，约4-8匹/月
- **月收入**：约6-16两（丰年），2-5两（灾年）

**关键经济触发阈值**：
- 月收入低于3两 → 生存线（吃紧）
- 月收入低于1两 → 危机线（要借债/卖机）
- 税负增加1两/月 → 从温饱跌到吃紧
- 织机损坏（3-5两修/换）→ 可能需要借高利贷

### 2.4 缺口：白银流动机制

万历年间江南经济的核心变量是白银流入：
- 海外白银经马尼拉-月港航线流入，每年约数百万两
- 白银到港季（5-8月）→ 丝价上涨，织户收入增加
- 白银断流（倭寇/海禁）→ 丝价下跌，织户收入骤降
- 一条鞭法后所有赋税折银 → 织户必须卖绸换银交税

**游戏化建议**：在era.json中增加 `silver_inflow` 变量（白银流入量），与行情周期联动：
- 高流入 → 丝价涨、牙行收购积极、织户收入增加
- 低流入 → 丝价跌、牙行压价、织户收入减少

## 三、官僚系统：需要补充什么

### 3.1 现有覆盖

era.json v1.1 已有：
- `court_paralysis`（朝廷失能程度）
- `tribute_pressure`（上供压力）

### 3.2 缺口：基层官僚层级

玩家是织户，接触不到皇帝和内阁，但天天和这些人打交道：

**织户会接触的官僚层级**（从近到远）：

| 层级 | 角色 | 与织户的关系 | 打点成本 |
|------|------|------------|---------|
| **里甲** | 里长、甲首、老人 | 征税、摊派、调解纠纷 | 几钱到1两 |
| **县衙吏员** | 书吏、皂隶、禁子 | 具体办事、传唤、收税 | 1-5两 |
| **县官** | 知县（正七品）、县丞（正八品）、主簿（正九品） | 审案、批文、加税 | 10-50两 |
| **府级** | 知府、推官 | 上诉、重审 | 50-200两 |
| **织造局** | 织造太监（内监系统） | 加征织造税、征用织户 | 无底洞 |
| **钞关** | 钞关主事（户部差遣） | 过往货物收税 | 货值的3-10%行贿 |

**关键规则**：
- **“衙门口朝南开，有理无钱莫进来”**：打官司必须先交“规费”（挂号费、传票费等），约1-3两
- **书吏是真正的办事人**：知县三年一换，书吏世袭本地，不打通书吏，知县批了也执行不了
- **里长是征税第一道关**：里长可以“通融”赋税缴纳时间，也可以“优先”催你
- **织造太监不受地方管辖**：内监系统直通宫中，地方官管不了

### 3.3 缺口：打点成本表

玩家遇到官府相关触发事件时，需要知道“花多少钱能办什么事”：

| 事项 | 打点对象 | 成本 | 效果 |
|------|---------|------|------|
| 催税缓缴 | 里长 | 0.5-1两 | 延期1-3月 |
| 免除杂派 | 书吏 | 2-5两 | 减免当年杂派 |
| 官司脱身 | 书吏+知县 | 10-30两 | 撤案或轻判 |
| 钞关逃税 | 钞关主事 | 货值3-5%行贿 | 实缴税额降至应缴10% |
| 织造局免征 | 织造太监手下 | 5-20两 | 暂免加征 |
| 买官（不入流） | 吏部 | 100-200两 | 典史、驿丞等小官 |
| 买官（正九品） | 吏部+中间人 | 300-500两 | 主簿 |

### 3.4 缺口：官府纠纷处理规则

| 纠纷类型 | 处理流程 | 时间 | 成本 |
|---------|---------|------|------|
| 织户与牙行 | 先找里长调解→不成去县衙 | 1-3月 | 2-10两 |
| 织户与客商 | 县衙起诉→知县判决 | 3-6月 | 5-20两 |
| 被人诬告 | 先被收押→找人保释→打官司 | 即时收押+1-6月 | 保释5-10两+官司10-30两 |
| 抗税 | 里长上报→县衙派差役→可能升级 | 即时 | 0（硬抗）或倾家荡产 |

## 四、两个系统如何联动

经济系统和官僚系统的联动，正是触发事件的核心驱动力：

| 联动场景 | 经济变量 | 官僚变量 | 触发事件 |
|---------|---------|---------|---------|
| 白银流入减少 → 丝价下跌 → 交不起税 | silver_inflow↓ | tax_burden不变 | 被催税→找里长通融或抗税 |
| 织造加税 → 利润归零 | tax_burden↑ | eunuch加征 | 生存危机→抗税/逃亡 |
| 钞关逃税 → 竞争对手成本更低 | 竞争对手chaoguan=0 | customs_bribe | 你不逃税就竞争不过 |
| 官司缠身 → 倾家荡产 | 储蓄清零 | dispute_cost | 被迫借高利贷→恶性循环 |
| 买官 → 用权力反哺生意 | 一次性大额支出 | 获得官身 | 从被管的人变成管人的人 |

**核心设计原则**：
1. 经济系统提供“数字感”——玩家知道一两银子能干什么
2. 官僚系统提供“规则感”——玩家知道遇到官府该怎么办
3. 两者联动产生“困境感”——交不起税→找官府通融→花更多钱→更交不起税

## 五、补充优先级

| 优先级 | 内容 | 理由 |
|--------|------|------|
| **P0** | 价格锚点体系 | 没有锚点，所有经济触发事件都是空谈 |
| **P0** | 打点成本表 | 没有成本表，官府触发事件无法量化 |
| **P1** | 成本-收益模型 | 让玩家理解“织一匹绸赚多少” |
| **P1** | 基层官僚层级 | 让DM知道“玩家遇到官府时该找谁” |
| **P2** | 白银流动机制 | 增加经济系统的动态性 |
| **P2** | 纠纷处理规则 | 增加官僚系统的可玩性 |
