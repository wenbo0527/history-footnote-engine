## 🐛 关键问题与解决

### 问题 1：LangChain LLM 不能当函数调用
- **症状**: `LLM 调用失败: 'ChatAnthropic' object is not callable`
- **根因**: Coordinator 把 LLM 当 callable 函数调，但 LangChain 类需 `.invoke()`
- **解决**: 加 `_invoke_llm(prompt_ctx)` 方法，检测 `hasattr(llm, "invoke")` 分两路
- **测试**: 221 测试全过，30 回合真 LLM 端到端跑通

### 问题 2：extract_json_from_text 返回字符串
- **症状**: 解析 LLM 输出后 `not isinstance(llm_output, dict)` 报 str
- **根因**: `extract_json_from_text` 返回 JSON 字符串，需 `json.loads` 再用
- **解决**: `dm_tool.py` 加 `json.loads(json_str)` 二次解析
- **测试**: V28_179 通过

### 问题 3：Blueprint 期望硬编码回退
- **症状**: 真 LLM 走通但 chapter_history 拿到硬编码 title "且听下回分解 · 春蚕"
- **根因**: 第一次 mock 测试无 ```json``` 包裹，extract_json 失败
- **解决**: 测试用 markdown 包裹
- **教训**: extract_json_from_text 实际只支持 ```json 包裹 / 末尾 {} 块

### 问题 4：第 1 章 LLM 走通但第 2 章失败
- **根因**: Coordinator 把 LLM 当函数调（只在 smoke_v280_real_llm.py 调成功是因为绕过了 Coordinator）
- **解决**: 同问题 1（_invoke_llm 兼容）

### 问题 5：KnowledgeBase() 缺参数
- **症状**: dm_agent Tool 测试 `TypeError: KnowledgeBase.__init__() missing 1 required positional argument: 'entries'`
- **根因**: KB 构造需要 entries，但 smoke 用 `KnowledgeBase()`
- **解决**: 改用 `MagicMock()` 模拟
